## UX/UI 원칙

### DP-001: 2단계 페이지 구조 (이미지 편집 도구)
**상태**: ✅ 채택됨  
**적용 대상**: Background Blur, Background Remove 등 이미지 편집 도구  
**최종 결정일**: 2025-12-03

#### 컨텍스트
이미지 편집 도구의 UI를 설계할 때 다음 두 가지 방식을 고려했습니다:

1. **2단계 페이지 구조** (현재 방식)
   - 1단계: 도구 설명 + 이미지 업로드 영역
   - 2단계: Original/Result 분할 화면 + 옵션 조절 + 다운로드

2. **직접 시작 방식** (대안)
   - 한 페이지에서 모든 것 표시
   - Original 영역에서 바로 이미지 선택 가능

#### 결정
**2단계 페이지 구조를 채택하고 유지합니다.**

#### 이유

##### 1️⃣ 사용자 경험 (UX) 측면
- ✅ **명확한 첫 행동 제시**: 사용자가 페이지 진입 시 "지금 해야 할 일"(이미지 선택)을 직관적으로 이해
- ✅ **심리적 진입장벽 낮음**: 초기 페이지가 간결하고 가벼워서 부담 없이 시작 가능
- ✅ **자연스러운 단계 전환**: 이미지 선택 → 편집 화면의 흐름이 논리적
- ✅ **초보자 친화적**: 복잡한 UI를 한 번에 보여주지 않아 혼란 방지

##### 2️⃣ 검색 엔진 최적화 (SEO) 측면
- ✅ **키워드 노출 극대화**: 첫 페이지에 "Background Blur", "Blur background of portrait photos" 등 핵심 키워드가 크게 표시됨
- ✅ **색인 우선순위 상승**: 검색 봇이 페이지 진입 시 바로 관련 키워드를 만나서 관련성 인식 향상
- ✅ **메타 설명 활용**: 도구 설명문이 검색 결과의 스니펫으로 활용되어 클릭률(CTR) 증대
- ✅ **콘텐츠 구조 명확성**: H1 제목과 설명이 페이지 상단에 위치하여 검색엔진이 주제를 명확히 인식

##### 3️⃣ 비즈니스 측면
- ✅ **전환율 최적화**: 명확한 CTA(Call-to-Action)로 사용자의 다음 행동 유도
- ✅ **브랜드 일관성**: 모든 이미지 도구가 동일한 패턴을 따라 학습 비용 감소

#### 대안의 단점 (직접 시작 방식을 선택하지 않은 이유)
- ❌ **첫 행동 모호**: "Original 영역을 클릭하라"는 명시적 안내 필요
- ❌ **초기 진입장벽 상승**: 분할 화면, 슬라이더, 버튼 등으로 복잡한 느낌
- ❌ **SEO 약화**: 검색 봇이 이미지 프리뷰나 UI 컨트롤에 먼저 노출되어 핵심 키워드 색인 약화
- ❌ **메타 설명 부실**: 도구 설명이 충분히 노출되지 않아 검색 결과 클릭률 감소
- ❌ **신규 사용자 이탈 위험**: 무엇을 해야 할지 모르겠다는 느낌으로 이탈 가능성 증가

#### 영향
- 모든 이미지 편집 도구(Background Blur, Background Remove, Image Resize 등)는 이 원칙을 따릅니다.
- 향후 새로운 이미지 도구 추가 시에도 동일한 2단계 구조를 적용합니다.
- 기존 도구의 UX 개선 시 이 구조를 변경하지 않습니다. (슬라이더 범위 조정, 옵션 추가 등은 가능)

#### 예외 사항
PDF 도구는 이 원칙의 적용 대상이 아닙니다. PDF는 이미지와 다른 UX 패턴을 따릅니다.
