## 1. PDF Tools 페이지 키워드 전략

### 현재 문제
- H1: "Free Online PDF Tools"
- 부제: "Securely convert, merge, and split PDFs in your browser"
- 문제: 너무 일반적, 강력한 경쟁사(Adobe, SmallPDF)와 직면

### 개선 방안

#### A. 주력 키워드 정하기
**타겟 검색 쿼리:**
- "무료 PDF 도구" (broad, 경쟁 높음)
- "온라인 PDF 변환" (medium, 경쟁 중간)
- "PDF 병합 무료" (long-tail, 경쟁 낮음) ⭐
- "PDF 분할 온라인" (long-tail, 경쟁 낮음) ⭐

**추천:** Long-tail 키워드에 집중 (경쟁 낮고 전환율 높음)

#### B. H1 & 메타 설명 개선

**현재:**
```
H1: Free Online PDF Tools
Meta: Securely convert, merge, and split PDFs in your browser
```

**개선:**
```
H1: Free Online PDF Tools - Convert, Merge & Split PDFs
Meta: 무료 PDF 병합, 분할, 변환 도구. 8가지 기능으로 PDF를 온라인에서 빠르게 처리하세요.
```

**이유:** 
- H1에 핵심 액션(Convert, Merge, Split) 명시
- 메타에 숫자("8가지") 포함 → 검색 결과에서 눈에 띔

#### C. 페이지 본문 구조 추가

현재는 도구 카드만 있으니, 이 아래에 추가 섹션:

```
## 왜 Lokit PDF Tools인가?

[100% client-side, Privacy first, Unlimited free 세 가지 강조]

## 어떤 PDF 작업이 필요한가요?

- PDF를 JPG/PNG로 변환하려면 → PDF → JPG
- 여러 PDF를 하나로 합치려면 → Merge PDFs
- 큰 PDF 파일을 줄이려면 → Compress PDF
- [등등...]

## PDF Tools 상세 설명

각 도구별 1-2문장 설명 추가
- PDF → JPG: "PDF를 JPG 이미지로 변환하세요..."
- Merge PDFs: "여러 PDF 파일을 한 번에 병합..."
```

## 2. Image Tools 페이지 키워드 전략

### 현재 문제
- H1: "Image Tools"
- 부제: "Edit quickly in your browser"
- 문제: 너무 추상적, 검색 의도 불명확

### 개선 방안

#### A. 주력 키워드 정하기
**타겟 검색 쿼리:**
- "무료 이미지 편집" (broad)
- "배경 흐림 온라인" (medium) ⭐
- "사진 배경 제거 무료" (medium) ⭐
- "이미지 크기 변경" (medium)
- "사진 압축 온라인" (long-tail) ⭐

**추천:** 이미지 도구는 medium + long-tail 모두 노리기 (검색량도 있고 경쟁도 적음)

**현재:**
```
H1: Image Tools
Meta: Edit quickly in your browser
```

## 4. 구조화된 데이터(Schema) 마크업

### ItemList 스키마란?
Google, Bing 등의 검색 엔진에 "이 페이지에는 여러 항목의 목록이 있다"는 걸 명시적으로 알려주는 코드입니다. 그러면 검색 결과에서 **리치 스니펫**(rich snippet)으로 표시됩니다.

### 현재 상태
Image Tools 페이지에 6개 도구가 나열되어 있지만, 검색 엔진은 이게 "목록"이라는 걸 모름.

### 개선 방안

#### A. ItemList 스키마 추가

Image Tools 페이지 HTML의 `<head>` 또는 도구 섹션에 다음 코드 추가:

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "ItemList",
  "name": "Free Online Image Tools",
  "description": "Edit quickly in your browser",
  "numberOfItems": 6,
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Background Blur",
      "description": "Blur background for portraits",
      "url": "https://lokit.tools/image/blur-background"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Background Remove",
      "description": "Remove image backgrounds",
      "url": "https://lokit.tools/image/background-remove"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Image Resize",
      "description": "Change dimensions safely",
      "url": "https://lokit.tools/image/resize"
    }
    [... 나머지 도구들]
  ]
}
</script>
```

#### B. 검색 결과 효과

**마크업 전:**
```
Image Tools
Edit quickly in your browser
lokit.tools/image
```

**마크업 후:**
```
Image Tools - 6 Free Online Image Editing Tools
Edit quickly in your browser
• Background Blur
• Background Remove
• Image Resize
• Image Compress
• Format Convert
• Image Crop
lokit.tools/image
```

### C. PDF Tools에도 동일하게 적용

PDF Tools 페이지에도 ItemList 스키마로 8개 도구를 마크업하면, 검색 결과에서 도구 목록이 한눈에 보입니다.

### 장점
- ✅ 검색 결과에서 눈에 띔 (CTR 증가)
- ✅ 사용자가 도구를 클릭하기 전에 어떤 도구가 있는지 확인 가능
- ✅ 검색 엔진이 페이지 구조를 명확히 이해 → SEO 향상

## 5. 상세 콘텐츠 추가 전략

현재 Lokit은 도구 카드만 있는데, 검색 사용자들의 의도를 만족시키려면 **설명 콘텐츠**가 필요합니다.

### A. "PDF Tools가 필요한 이유" 섹션

**위치:** PDF Tools 페이지의 도구 카드 **위**에 추가

**목적:** 사용자가 "왜 이 도구들이 필요한가"를 이해하도록

**예시 콘텐츠:**

```
## PDF 작업이 어렵나요?

많은 사람들이 PDF 작업 때문에 고민합니다:

• 여러 PDF를 하나로 합쳐야 하는데 방법을 모르겠어요
• 큰 PDF 파일이 이메일로 전송이 안 돼요
• PDF에서 특정 페이지만 추출하고 싶어요
• PDF를 JPG 이미지로 변환해야 하는데...

Lokit이 모든 문제를 해결합니다.

## Lokit PDF Tools의 장점

✅ **설치 불필요** - 브라우저만 있으면 바로 사용
✅ **완전 무료** - 회원가입, 숨은 비용 없음
✅ **완벽한 보안** - 모든 작업이 당신의 기기에서만 진행 (서버 업로드 X)
✅ **빠른 속도** - 복잡한 PDF도 몇 초 안에 처리
```

**검색 유입 효과:**
- "PDF 병합 방법", "PDF 압축 무료" 같은 검색에서 노출 증가
- 사용자가 문제를 느끼는 단계부터 Lokit을 알게 됨

---

### B. "어떤 도구를 언제 사용하는가" 가이드

**위치:** 각 도구 카드 **아래**에 추가 (또는 별도 섹션)

**목적:** 사용자가 자신의 상황에 맞는 도구를 찾도록

**예시 콘텐츠:**

```
## 당신의 상황에 맞는 도구 선택하기

### 상황 1: "PDF를 이미지로 변환해야 해"
→ **PDF → JPG** 또는 **PDF → PNG** 도구 사용
예: 스캔한 문서를 이미지로 변환, SNS에 업로드하려면

### 상황 2: "여러 PDF를 하나로 합쳐야 해"
→ **Merge PDFs** 도구 사용
예: 여러 페이지의 계약서, 보고서를 한 파일로

### 상황 3: "PDF 파일이 너무 커서 이메일 전송이 안 돼"
→ **Compress PDF** 도구 사용
예: 스캔 문서, 고해상도 이미지 포함 PDF 축소

### 상황 4: "PDF에서 특정 페이지만 빼야 해"
→ **Split PDF** 도구 사용
예: 10페이지 중 1-3페이지만 필요할 때

### 상황 5: "PDF 텍스트를 복사해야 하는데 이미지야"
→ **PDF → Text** 도구 사용
예: 스캔한 문서에서 텍스트 추출

### 상황 6: "PDF 페이지 방향이 잘못됐어"
→ **Rotate PDF** 도구 사용
예: 가로/세로 방향 수정

### 상황 7: "사진들을 PDF로 만들고 싶어"
→ **Image → PDF** 도구 사용
예: 여러 사진을 하나의 PDF 문서로
```

**검색 유입 효과:**
- "PDF 파일 이미지로 변환하는 법", "PDF 여러 개 합치기" 같은 **How-to 검색**에서 노출
- 사용자의 실제 필요를 해결하는 콘텐츠 → 높은 체류 시간 → SEO 향상

### C. "각 도구 비교" 가이드 (선택사항)

**위치:** PDF Tools 또는 Image Tools 페이지 하단

**목적:** 비슷한 도구들의 차이를 명확히 하기

**예시 (Image Tools의 경우):**

```
## 이미지 편집 도구 비교

| 필요한 작업 | 추천 도구 | 특징 |
|---|---|---|
| 배경을 흐리게 하기 | Background Blur | 사람을 살리고 배경 흐림 |
| 배경 완전히 제거 | Background Remove | 투명 배경으로 변환 |
| 이미지 크기 조정 | Image Resize | SNS 규격에 맞추기 |
| 파일 크기 줄이기 | Image Compress | 품질 유지하며 압축 |
| 형식 변경 (JPG↔PNG) | Format Convert | PNG→JPG, JPG→WebP 등 |
| 사진 잘라내기 | Image Crop | 불필요한 부분 제거 |

### Background Blur vs Background Remove?

**Background Blur:** 배경을 "흐리게" 함
→ 사용: 프로필 사진, 초상화, 증명사진
→ 배경이 완전히 제거되지 않음 (더 자연스러움)

**Background Remove:** 배경을 "완전히 제거" (투명)
→ 사용: 제품 사진, 투명 배경 필요할 때
→ 배경이 없는 이미지 (PNG 투명 배경)
```

**검색 유입 효과:**
- "배경 흐림과 배경 제거 차이", "어떤 도구를 써야 할까" 같은 비교 검색 유입
- 사용자가 혼동하는 개념을 명확히 → 신뢰도 증가


### D. 내부 링크/백링크 가이드 (가시성 중심)

- 위치: 1) 페이지 상단 브레드크럼(홈 > 카테고리 > 도구), 2) 본문 중간 "이 작업 끝나면 같이 쓰는 도구" 카드 리스트 3~6개, 3) 본문 하단 카테고리 전체 보기 CTA.
- 앵커 텍스트: 카테고리/도구 키워드를 포함하고 숨김·미니멀 링크는 피함, 밑줄/색상으로 클릭 가능 표시.
- 레이아웃: 카드에 아이콘/썸네일 + 한 줄 설명 추가, 버튼/텍스트를 폴드 위·중간·하단에 분산해 CTR 개선.
- 측정: Search Console 내부 링크 보고서와 페이지 CTR을 1~2주 단위로 확인.

## 6. 적용 우선순위

### Phase 1 (필수)
1. ✅ 각 페이지의 H1 & 메타 설명 업데이트
2. ✅ Schema ItemList 마크업 추가 (PDF/Image Tools)
3. 🔄 개별 도구 페이지 보이는 내부 링크 배치(브레드크럼 + 관련 도구 카드 + 카테고리 CTA)

### Phase 2 (권장)
1. 각 카테고리 페이지에 본문 콘텐츠 추가
2. 개별 도구 페이지에 "사용 사례" 섹션 추가
3. How-to 성격의 상세 가이드 페이지 작성(튜토리얼/스크린샷/팁, 롱테일 키워드 타겟, 도구 CTA 포함)
4. 개별 도구 페이지 ↔ 가이드 페이지 양방향 내부 링크 배치(도구 → "처음이면 가이드 보기" 텍스트 링크, 가이드 → 도구 CTA)

### Phase 3 (심화)
1. 각 도구별 블로그 가이드 작성
2. FAQ 섹션 추가

## 8. 예상 SEO 효과

### 단기 (1-2주)
- H1/메타 개선으로 검색 관련성 향상
- Schema 마크업 적용 시작

### 중기 (1-2개월)
- 검색 결과 Rich Snippet 노출 시작
- Long-tail 키워드 순위 상승
- CTR (클릭률) 증가

### 장기 (3-6개월)
- 주요 키워드 순위 안정화
- 자연 유입 트래픽 증가
- 브랜드 인지도 향상
